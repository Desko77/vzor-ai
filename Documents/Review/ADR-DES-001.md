# ADR-DES-001: Voice UI состояния и управление голосовой сессией

**Статус:** Proposed\
**Дата:** 2026-03-04

## Контекст

UI прототип приложения Vzor включает экраны:

-   Home
-   Conversation
-   History
-   Settings
-   Logs

Однако не определено как интерфейс отражает состояния голосового
ассистента.

Для voice‑first интерфейса пользователь должен видеть в каком состоянии
находится ассистент.

## Решение

UI синхронизируется с состояниями VoiceOrchestrator.

Добавляется компонент **Voice State Indicator**.

Состояния:

Listening\
Thinking\
Speaking\
Offline\
Error

## Поведение интерфейса

### Listening

-   отображается waveform
-   показывается partial transcript

### Thinking

-   отображается индикатор обработки
-   блокируется повторная активация

### Speaking

-   отображается активный ответ
-   доступна кнопка Stop

### Offline

-   показывается badge Offline
-   ассистент использует локальные модели

## Confirmation UI

Для действий требуется подтверждение.

Пример:

Отправить сообщение Саше?

Кнопки:

Отправить\
Отмена

## Barge‑in

Если пользователь начинает говорить во время ответа:

-   TTS останавливается
-   состояние → LISTENING

## Альтернативы

### Только голосовые подтверждения

Отклонено.

Причина:

визуальное подтверждение снижает ошибки.

### Не отображать состояния

Отклонено.

Причина:

пользователь не понимает что происходит.

## Последствия

Плюсы:

-   понятный UX
-   меньше ошибок

Минусы:

-   больше UI состояний
-   требуется синхронизация с Orchestrator
